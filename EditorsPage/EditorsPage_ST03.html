Feedback Page

Technical Description

The Feedback Page is made using HTML, CSS, and JavaScript to create a simple and easy-to-use form where users can give their feedback. The form includes text boxes, email fields, dropdowns, radio buttons, checkboxes, and a rating system to collect different types of feedback.

CSS helps make the page neat, well-spaced, and responsive, so it works well on different screen sizes. JavaScript checks the inputs and shows error messages if something is missing or incorrect. The submit button ensures that users complete the form properly before sending it.

After submitting the form successfully, a message appears to confirm submission, making the process clear for users. Smooth animations and transitions make the page look modern and user-friendly.

Accessibility

The Feedback Page follows good accessibility practices so that everyone, including people with disabilities, can use it easily. The form uses proper labels for input fields to help screen readers read the information correctly.

  • Color Contrast: High contrast between text and background improves readability.
  • Keyboard Navigation: Users can navigate through the form using the keyboard, enhancing accessibility for those with motor impairments.
  • ARIA Attributes: Implemented where necessary to improve screen reader interpretation.
  • Validation Messages: Clear, descriptive error messages assist users in correcting input errors.

These features make the Feedback Page easy, accessible, and user-friendly for everyone.

Team Page

Technical Description

The HTML and CSS techniques used in the Team Page focus on a clean, user-friendly design with interactivity and accessibility.

Key HTML Tags:

  1. <section>
    • The <section class="team-container"> wraps the entire content of the team page. It groups related content for better semantics, improving readability and maintainability.

  2. <div class="team-grid">
    • A grid container that holds all the team members' cards. It provides a flexible layout for organizing team member information.

  3. <div class="team-member">
    • This represents each individual team member's card. Inside it, there are images and detailed information (name, role, tasks) about the member. The tabindex="0" allows the cards to be focusable for accessibility (keyboard navigation).

  4. <h3> and <p>
    • <h3> tags are used for team member names, providing a clear hierarchy of content.
    • <p> tags present additional information like the role and tasks of each member.

  5. <a>
    • Links used for navigation throughout the page and the footer. The links are styled to match the theme of the page and provide users with clear call-to-actions (CTA).

Key CSS Properties:

  1. Colors:
    • The main colors used are green for primary buttons, light green for secondary elements, and white for backgrounds. These colors make the page look fresh and clean.

  2. Fonts:
    • The page uses a modern font called 'Inter' for a professional and readable look.

  3. Team Grid:
    • The team members are arranged in a 2-column grid. It adjusts to a single column on small screens (like phones).
    • The grid helps make the page look organized and neat.

  4. Team Member Cards:
    • Each team member is in a card with a border, rounded corners, and a shadow to make it stand out.
    • When you hover over the card, it gets slightly bigger and the shadow becomes more intense. This makes it feel interactive.

  5. Team Member Images:
    • The photos of the team members are circles with a border around them. On hover, the images get a bit bigger to grab attention.

  6. Details Display:
    • When you hover over a team member card, more details (like their tasks) appear. This keeps the page clean and only shows more info when needed.

Accessibility

The Team Page is designed with accessibility in mind to ensure a smooth and inclusive experience for all users, including those with disabilities. The following accessibility features are implemented:

  1. Keyboard Navigation & Focus Management
    • Tab Navigation:
      • All interactive elements (like team member cards and navigation links) are keyboard-accessible.
      • tabindex="0" is applied to team member cards, making them focusable for users navigating via keyboard.
    • Skip Links:
      • A "Skip to Main Content" link helps users jump past the menu to the main page.
    • Highlighting Selected Items:
      • When you use the keyboard to navigate, the selected item is clearly highlighted

  2. Screen Reader Compatibility
    • Good Page Structure:
      • The page uses proper headings (<h3>, <p>) and sections (<nav>, <footer>) so screen readers can read it correctly.
    • Image Descriptions (Alt Text):
      • All images have descriptions so visually impaired users know what they are.

  3. Contrast & Readability;
    • High Contrast Colors:
      • Dark text on a light background ensures readability for users with low vision or color blindness.
    • Resizing Text Without Breaking Layout:
      • Users can increase text size up to 200% without content overlap or loss.
    • Legible Fonts & Spacing:
      • The ‘Inter’ font is used for clarity.
      • Sufficient line height and letter spacing improve readability.

Content Page

Technical Description

The Content Page is designed with a structured layout, ensuring clear information presentation, accessibility, and a responsive design. The page utilizes HTML for content organization and CSS for styling, creating a visually appealing and user-friendly experience.

Key HTML Tags:

  1. <section> (Main Content Wrapper)
    • The <section class="content-container"> acts as the main wrapper for the content page.
    • Enhances page structure and semantics for better readability and accessibility.

  2. <div class="content-grid"> (Grid Layout for Content Sections)
    • Organizes content into a flexible grid layout, ensuring a structured design.
    • Adapts to different screen sizes for responsive behavior.

  3. <img> (Images in Content Sections)
    • Used to enhance the visual appeal of the content.
    • alt attributes are provided to improve accessibility for visually impaired users.

Key CSS Properties:

  1. Colors:
    • The primary color scheme consists of green accents for buttons, light backgrounds for readability, and white space for a clean layout.
    • Ensures visual clarity and maintains consistency across the website.

  2. Typography:
    • The page uses ‘Inter’ as the main font for a modern and professional look.
    • Headings (h2, h3) have larger font sizes to create a clear content hierarchy.
    • Line height and letter spacing are optimized for readability.

  3. Grid Layout:
    • Uses CSS Grid to arrange content sections in a structured format.
    • The grid layout adapts to a single-column view on smaller screens for better mobile compatibility.

  4. Interactive Elements:
    • Hover Effects: When hovering over content blocks, a subtle shadow effect appears, making the sections stand out.
    • Button Styling: have rounded edges and color changes on hover, making them visually appealing and interactive

  5. Content Images:
    • Circular Borders: Images within articles are styled with rounded borders for a sleek look.
    • Hover Zoom Effect: slight enlargement on hover to grab attention.

Accessibility

Explain how you used to addressed accessibility in this particular page you implemented.

  1. Keyboard Navigation & Focus Management
    • Tab Navigation:
      • All interactive elements, including links and buttons, are keyboard-accessible.
      • tabindex="0" is applied to key content blocks, allowing users to focus on them using the keyboard.

    • Highlighting Focused Elements:
      • When using the keyboard to navigate, focused elements are visibly highlighted, ensuring users can track their position.

  2. Screen Reader Compatibility
    • Proper Page Structure:
      • Uses appropriate headings (h2, h3), sections (<article>), and landmarks (<nav>, <footer>) for screen reader compatibility.

    • Image Descriptions (alt Text):
      • All images include descriptive alt text, enabling visually impaired users to understand the content

  3. Contrast & Readability
    • High Contrast Colors:
      • Text contrasts well against the background, making it readable for users with low vision or color blindness.

    • Text Resizing Without Layout Breakage:
      • Users can increase text size up to 200% without causing content overflow or layout distortion.

    • Legible Fonts & Spacing:
      • Sufficient line spacing and letter spacing enhance readability.
