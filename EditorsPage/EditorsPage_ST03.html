Feedback Page

Technical Description

The Feedback Page is designed using HTML, CSS, and JavaScript to create a user-friendly form for collecting feedback. The form includes various input types such as text fields, email fields, dropdowns, radio buttons, checkboxes, and a rating system to gather comprehensive feedback from users.

Key HTML Tags:

  1. Form Tag (<form>)
    • The <form>: tag is used to create the feedback form. It acts as a container for all the input fields and buttons.

  2. Input Fields (<input>, <textarea>):
    • <input>: Used for single-line inputs like name and email. The type attribute specifies the kind of input (e.g., text, email)
    • <textarea>: Used for multi-line inputs like feedback messages. It allows users to write longer responses.

  3. Labels (<label>):
    • Each input field has a <label> tag to describe its purpose. The for attribute connects the label to the input field’s id.

  4. Buttons (<button>):
    • The <button> tag is used for actions like submitting the form. The type="submit" attribute sends the form data.

  5. Divisions (<div>):
    • The <div> tag groups related elements together, making it easier to style and organize the form.

Key CSS Properties:

  1. Form Layout:
    • Used display: flex or display: grid to arrange the form elements neatly. For example, stack input fields vertically and center the form on the page.

  2. Styling Input Fields:
    • Input fields are styled with padding, border, and border-radius to create a clean and modern look. The width: 100% ensures they fill the container.

  3. Hover and Focus States:
    • Add effects when users hover over or click on an input field. For example, change the border color when the field is focused.

  4. Button Styling:
    • Style the submit button to match the theme of the page. Add hover effects to make it interactive.

  5. Responsive Design:
    • Use media queries to adjust the form layout for smaller screens. For example, reduce the width of input fields on mobile devices.

JavaScript Functionality:

  1. Form Validation:
    • Added JavaScript to validate the form before submission. For example, checked if the email field contains a valid email address and if the feedback message is not empty.

  2. Displaying Success Message:
    • After the form is submitted successfully, used JavaScript to display a success message to the user.

  3. Rating Emoji Selection:
    • Allows users to select a rating by clicking on emojis.
    • Stores the selected rating value in a hidden input field for form submission.

Accessibility

The Feedback Page is designed with accessibility in mind to ensure inclusivity for all users, including those with disabilities.

  1. Keyboard Navigation & Focus Management
    • Labels and Inputs:
      • Ensured every input field has <label> tag with a for attribute. This makes the form more accessible to screen readers and keyboard users.
    • Focus Indicators:
      • Added clear outlines or borders to focused elements so keyboard users can easily navigate the form.

  2. Screen Reader Compatibility
    • Semantic HTML:
      • Used proper tags like <form>, <label>, and <button> to ensure screen readers can understand and navigate the form.

  3. Contrast & Readability;
    • High Contrast Colors:
      • Dark text on a light background ensures readability for users with low vision or color blindness.
    • Legible Fonts and Spacing:
      • chose a clean font ('Inter') and added enough spacing between lines and letters for readability.

Team Page

Technical Description

The Team Page is designed using HTML and CSS to showcase team members in a clean and interactive layout. The page features a grid of team member cards, each containing an image, name, role, and additional details.

Key HTML Tags:

  1. Section Tag (<section>)
    • The <section class="team-container"> wraps the entire content of the team page. It groups related content for better semantics, improving readability and maintainability.

  2. Grid Container (<div class="team-grid">)
    • A grid container that holds all the team members' cards. It provides a flexible layout for organizing team member information.

  3. Team Member Cards (<div class="team-member">)
    • This represents each individual team member's card. Inside it, there are images and detailed information (name, role, tasks) about the member. The tabindex="0" allows the cards to be focusable for accessibility (keyboard navigation).

  4. Headings and Paragraphs(<h3> and <p>)
    • <h3> tags are used for team member names, providing a clear hierarchy of content.
    • <p> tags present additional information like the role and tasks of each member.

  5. Links (<a>)
    • Links used for navigation throughout the page and the footer. The links are styled to match the theme of the page and provide users with clear call-to-actions (CTA).

Key CSS Properties:

  1. Colors:
    • The main colors used are green for primary buttons, light green for secondary elements, and white for backgrounds. These colors make the page look fresh and clean.

  2. Fonts:
    • The page uses a modern font called 'Inter' for a professional and readable look.

  3. Team Grid:
    • The team members are arranged in a 2-column grid. It adjusts to a single column on small screens (like phones).
    • The grid helps make the page look organized and neat.

  4. Team Member Cards:
    • Each team member is in a card with a border, rounded corners, and a shadow to make it stand out.
    • When you hover over the card, it gets slightly bigger and the shadow becomes more intense. This makes it feel interactive.

  5. Team Member Images:
    • The photos of the team members are circles with a border around them. On hover, the images get a bit bigger to grab attention.

  6. Details Display:
    • When you hover over a team member card, more details (like their tasks) appear. This keeps the page clean and only shows more info when needed.

Accessibility

The Team Page is designed with accessibility in mind to ensure a smooth and inclusive experience for all users, including those with disabilities.

  1. Keyboard Navigation & Focus Management
    • Tab Navigation:
      • All interactive elements (like team member cards and navigation links) are keyboard-accessible.
      • tabindex="0" is applied to team member cards, making them focusable for users navigating via keyboard.
    • Highlighting Selected Items:
      • When you use the keyboard to navigate, the selected item is clearly highlighted

  2. Screen Reader Compatibility
    • Good Page Structure:
      • Proper tags like <sections>, <h3>, and <p> are used to ensure screen readers can interpret the content.
    • Image Descriptions (Alt Text):
      • All images include descriptive alt text for visually impaired users.

  3. Contrast & Readability;
    • High Contrast Colors:
      • Dark text on a light background ensures readability for users with low vision or color blindness.
    • Resizing Text Without Breaking Layout:
      • Users can increase text size up to 200% without content overlap or loss.
    • Legible Fonts & Spacing:
      • The ‘Inter’ font is used for clarity.
      • Sufficient line height and letter spacing improve readability.

Content Page

Technical Description

The Content Page is designed with a structured layout, ensuring clear information presentation, accessibility, and a responsive design. The page utilizes HTML for content organization and CSS for styling, creating a visually appealing and user-friendly experience.

Key HTML Tags:

  1. <section> (Main Content Wrapper)
    • The <section class="content-container"> acts as the main wrapper for the content page.
    • Enhances page structure and semantics for better readability and accessibility.

  2. <div class="content-grid"> (Grid Layout for Content Sections)
    • Organizes content into a flexible grid layout, ensuring a structured design.
    • Adapts to different screen sizes for responsive behavior.

  3. <img> (Images in Content Sections)
    • Used to enhance the visual appeal of the content.
    • alt attributes are provided to improve accessibility for visually impaired users.

Key CSS Properties:

  1. Colors:
    • The primary color scheme consists of green accents for buttons, light backgrounds for readability, and white space for a clean layout.
    • Ensures visual clarity and maintains consistency across the website.

  2. Typography:
    • The page uses ‘Inter’ as the main font for a modern and professional look.
    • Headings (h2, h3) have larger font sizes to create a clear content hierarchy.
    • Line height and letter spacing are optimized for readability.

  3. Grid Layout:
    • Uses CSS Grid to arrange content sections in a structured format.
    • The grid layout adapts to a single-column view on smaller screens for better mobile compatibility.

  4. Interactive Elements:
    • Hover Effects: When hovering over content blocks, a subtle shadow effect appears, making the sections stand out.
    • Button Styling: have rounded edges and color changes on hover, making them visually appealing and interactive

  5. Content Images:
    • Circular Borders: Images within articles are styled with rounded borders for a sleek look.
    • Hover Zoom Effect: slight enlargement on hover to grab attention.

Accessibility

The Content Page is designed to be accessible to all users, including those with disabilities.

  1. Keyboard Navigation & Focus Management
    • Tab Navigation:
      • All interactive elements, including links and buttons, are keyboard-accessible.
      • tabindex="0" is applied to key content blocks, allowing users to focus on them using the keyboard.

    • Highlighting Focused Elements:
      • When using the keyboard to navigate, focused elements are visibly highlighted, ensuring users can track their position.

  2. Screen Reader Compatibility
    • Proper Page Structure:
      • Uses appropriate headings (h2, h3), sections (<article>), and landmarks (<nav>, <footer>) for screen reader compatibility.

    • Image Descriptions (alt Text):
      • All images include descriptive alt text, enabling visually impaired users to understand the content

  3. Contrast & Readability
    • High Contrast Colors:
      • Text contrasts well against the background, making it readable for users with low vision or color blindness.

    • Text Resizing Without Layout Breakage:
      • Users can increase text size up to 200% without causing content overflow or layout distortion.

    • Legible Fonts & Spacing:
      • Sufficient line spacing and letter spacing enhance readability.

Challenges and Lessons Learned

While working on the Feedback Page, Team Page, and Content Page, I faced a few challenges. One challenge was aligning and managing content effectively on the pages. I used different styling techniques like CSS Grid and Flexbox to organize the layout, and I experimented with tags like <section>, <div>, and <article> to structure the content properly. Another challenge was coordinating with my team, as this was a team project. To manage our work efficiently, we used version control systems like Git, which helped us track changes, avoid conflicts, and collaborate smoothly.

From these challenges, I learned the importance of planning layouts carefully and using the right HTML tags and CSS properties to create a clean and organized design. I also gained valuable experience in teamwork and collaboration, which improved my ability to work effectively in a group setting. These experiences helped me grow as a developer and taught me the importance of clear communication and structured workflows.

Compliance

The pages I created (Feedback Page, Team Page, and Content Page) were designed with general best practices in mind, though they were not explicitly built to comply with Janet regulations. However, I ensured that the pages follow principles that align with common web standards, including accessibility, usability, and content organization.

For accessibility, I used semantic HTML tags like <header>, <section>, and <label to structure the content logically. I also added descriptive alt text for images and ensured keyboard navigability by using focus indicators and tabindex for interactive elements. These practices make the pages more inclusive and user-friendly.

In terms of content organization, I used CSS Grid and Flexbox to create clean and responsive layouts. This ensures that the pages are easy to navigate and visually appealing across different devices. I also used proper headings (<h1>, <h2>, etc.) to create a clear content hierarchy, which improves readability and helps users find information quickly.

I focused on creating a solid foundation for accessibility and usability. For full compliance with Janet regulations, additional steps like conducting accessibility audits, ensuring secure data handling, and adhering to specific guidelines would be necessary.
