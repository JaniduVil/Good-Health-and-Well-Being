User Profile Page

Technical Description

The User Profile Page is built using semantic HTML for clear content organization and external CSS for structured styling. The layout follows a grid-based design, ensuring a clean and visually appealing presentation.

Key HTML Elements Used
  • <nav>: Defines the navigation bar with a list of links for easy access.
  • <section>: Organizes profile details, skills, goals, and availability into structured segments.
  • <div class="profile-wrapper">: Wraps the profile image and user details for styling and alignment.
  • <div class="skills-list">: Displays skills and progress bars, enhancing visual clarity.
  • <table class="availability">: Structures availability data into a readable table format.
  • <form class="contact-form">: Provides an interactive contact section with input fields.
  • <footer">: Contains quick links and credits for easy navigation.
Key CSS Properties Used
  • display: grid; – Structures content using a responsive grid layout.
  • border-radius: 8px; – Softens card and button edges for a modern UI.
  • box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1); – Enhances depth and contrast.
  • :hover { transform: scale(1.05); } – Adds interactive hover effects for user engagement.

Accessibility

To ensure an inclusive and user-friendly experience, accessibility best practices were followed:

  • Semantic HTML elements improve screen reader navigation.
  • alt attributes are provided for all images
  • Text contrast ensures readability, with sufficient color differentiation.

Sitemap Page

Technical Description

The sitemap is implemented using HTML, CSS, and JavaScript. The structure is represented using SVG elements to create an interactive visual map of the website's navigation. The implementation ensures responsiveness, hover effects, and clickable nodes that redirect users to respective pages.

Key HTML Elements Used

The sitemap is placed inside a

to ensure proper layout. The main structure is built using SVG elements:

  • <g>: Groups elements (node + text) together for interactivity.
  • <rect>: Defines the shape of each page node.
  • <text>: Centers the text inside the node.
  • <line>: Creates connections between nodes.
Key CSS Properties Used
  • rx: – Make rounded corners in nodes.
  • fill: – Fill the background of the node.
  • text-anchor: middle – Center-aligned text.
  • media queries – Adapt Text sizes based on screen size.
Key JavaScript Properties Used
  • window.location.href – Ensures proper path structure for different directories.

Accessibility

  • Hover effects provide visual feedback.
  • Clickable areas ensure easy navigation.
  • Scalability allows adding new pages without breaking the design.

Content Page

Technical Description

The Preventive Healthcare page is designed with a clear, structured layout to ensure accessibility, responsiveness, and ease of navigation. Built using HTML for content structure and CSS for styling, the page offers a seamless and informative user experience on preventive healthcare. The design focuses on providing critical health information, resources, and interactive elements to educate users about preventive measures, screenings, and healthy lifestyle choices.

Key HTML Elements Used
  • <section>: Organizes content into sections, providing a clear structure for the page.
  • <div class="content-grid">: Organizes the page content into a flexible grid layout. Ensures that the page is responsive, adapting well to different screen sizes and devices.
  • <img>: Images are used to visually enhance content, such as infographics and health tips. All images include alt attributes to provide descriptions for visually impaired users.
Key CSS Properties Used
  • Colors: – The primary color scheme utilizes shades of blue and green for buttons and headings, with a light background color for readability.
  • Typography: – The page uses ‘Arial’ and ‘Roboto’ fonts for clarity and professionalism. Headings (h2, h3) are larger to establish a content hierarchy, while text size is optimized for readability.
  • Grid Layout: – The layout utilizes CSS Grid to arrange the content into structured, easy-to-navigate sections.
  • Hover Effects: – Interactive content sections and buttons have subtle hover effects, such as color changes, to provide visual feedback to users.

Accessibility

  • Focused elements are visibly highlighted when users navigate using the keyboard, helping users track their position on the page
  • The page structure uses appropriate HTML tags, including headings (h2, h3), article, and nav, for easy navigation by screen readers. Landmarks such as header, footer, and main ensure clear navigation for assistive technologies.
  • The fonts used on the page are clear and easy to read. Proper line spacing and letter spacing are applied to improve readability and make the content more digestible for users with cognitive impairments.

Challenges and Lessons Learned

  • Ensuring that the design, navigation, and functionality remain consistent across the profile page, sitemap, and content page, creating a unified user experience.
  • Making sure the pages adapt well to different screen sizes (desktop, tablet, mobile) and maintain a clean and usable layout.

Compliance

Creating a Profile Page, Sitemap, and Content Page requires compliance with accessibility guidelines and performance optimization. This ensures the pages are inclusive, secure, and user-friendly across devices, providing a smooth experience while meeting legal and technical standards.
